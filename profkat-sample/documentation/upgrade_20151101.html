MyCoRe Professorenkatalog: Aktualisierung der Anwendungen 2015-11-01

Warum?

  • Anpassung an aktuelle MyCoRe LTS Version (2015.05 LTS)

pom.xml

<parent>...</parent>

Konfigurationsdateien

  • werden ins resources-Verzeichnis verschoben, damit entfällt die Spezialbehandlung im Build-Prozess
  • verschiebe /src/main/config/lang/messages_*.properties nach /src/main/resources/messages*_*.properties
  • verschiebe /src/main/config/mycore.properties nach /src/main/resources/mycore.properties
  • verschiebe /src/main/config/web.xml nach /src/main/resources/config/web.xml
  • verschiebe /src/main/config/navigation.xml nach /src/main/resources/config/navigation.xml

web.xml

  • CHECK: Anwendungsname in <diplay-name>
    Dieser Name ist gleichzeitig der Name des Konfigurationsverzeichnisses im $user.home.

hibernate.cfg.xml

  • Löschen, wird automatisch im Konfig-Verzeichnis im $user.home erstellt und muss jetzt dort konfiguriert werden
  • siehe MyCoRe-Dokumentation

build.xml

  • Basis aus profkat-sample übernehmen und neu gem. Anleitung anpassen

Start der MyCoRe-Kommandozeile

  • MCR.AppName Property setzen !!!
  • cd /opt/profkat/projects/profkat-sample-webapp/target/app
  • sudo -u tomcat7 java -jar -Xmx128m -DMCR.AppName=profkat-sample -jar profkat-sample-cli.jar
